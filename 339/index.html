취미로 코딩하는 컴덕의 블로그

블로그 이사, DigitalOcean X WordPress

2014년 11월 5일 by ASeom Han

티스토리 블로그를 연 지 세달 가량 지나니 슬슬 아쉬운 것들이 많아지기 시작했다.
티스토리가 뛰어난 블로그 플랫폼이라는 것에는 동의하지만, Tumblr와 같은 외국의 블로그 플랫폼보다는 못하다.
세계적인 IT 트렌드와는 담을 쌓고있는 한국의 웹 서비스를 그닥 좋아하지 않는 편이었는데, 티스토리도 예외는 아니었던 모양.

티스토리의 스킨 기능은 강력한 편이라 블로그의 디자인에는 아쉬움이 없었지만, 문제는 다른 곳에 있었다.
블로그를 관리하는 입장에서 하루에서 수십번 들락거리기 마련인 관리 페이지가 썩 마음에 들지 않는다는 것. 이게 뭔 대수냐 싶겠지만, 본인이 블로그를 하는 이유는 순전히 ‘재미있어서’이니까 꽤 중요한 부분이다.
관리 페이지의 인터페이스들이 HiDPI를 지원하지 않아 내가 가진 기기들에서는 흐릿하게 나오고, Tumblr와 같은 외국의 블로그 플랫폼처럼 깔끔하지도 못하다. 온갖 이미지 버튼 좀 집어치우고 CSS 위주로 만들어주면 얼마나 좋을까.

이것보다도 더 중대한 문제가 있는데, 모바일에서 글을 올리는 것이 노가다의 연속이라는 것.
본인은 사정상 PC보다 모바일로 접속하는 경우가 압도적으로 많다.
그런데, 티스토리 앱을 이용하면 글을 HTML로 작성할 수 없다니 이게 뭔 멘붕오는 상황인가.
글자 색을 진회색으로 지정하는 태그를 맘대로 넣어버리는 것에 2차 멘붕.
이전에는 개인 개발자가 개발한 TistoryM 앱을 통해 HTML과 Markdown으로 작성이 가능했지만, 공식 앱의 출시 탓인지 해당 앱이 앱스토어에서 증발. 절망적이다.

결국 다소 무식한 방법을 선택할 수 밖에 없었다. 티스토리 앱에서 사진을 첨부해 대충 비공개로 글을 올리고, PC 버전으로 글 수정에 들어가 글의 HTML 소스를 Drafts 앱으로 가져와 마저 작성한 뒤, 그걸 다시 글 수정에 들어가 집어넣고, 발행으로 공개 설정을 변경한다.
이 일을 글 올릴 때 마다 반복해야 하니 괴로운건 당연한 것.

이런저런 이유로 새로운 블로그 플랫폼을 갈망하던 차에, WordPress가 생각났고, 비용이 좀 들더라도 설치형 블로그에 도전해보고 싶다는 욕심이 생겼다.
웹 호스팅 대신 DigitalOcean과 같은 VPS를 이용하면 나만의 Linux 서버를 구성할 수 있다는 것에 솔깃했고, 왠지 재미있을 것 같았다.
이 글을 보고 하루 정도 고민한 끝에 바로 작업에 들어갔다.

작업 시작

DigitalOcean 계정을 만들고, Ubuntu 14.04 + WordPress 서버를 싱가포르에 열었다. 월 $5의 플랜을 선택했는데, 가입시 받은 $10의 크레딧으로 두 달 간은 무료로 쓸 수 있다.

터미널에서 SSH로 진입해 root 계정의 암호를 변경한 뒤, 웹 브라우저로 접속한다. 이 때 또 다른 암호를 요구하는데, 터미널에 표시되는 것을 입력하면 된다.
그 후 지시에 따라 WordPress 설치를 진행한다.

서버는 준비되었는데, 도메인이 없다.
가비아에서 aseom.net 도메인을 구입. 1년에 14,000원을 내야 한다. 한달에 대략 천원이라 생각하면 그리 아깝지는 않다.
네임서버는 DigitalOcean이 제공하는 것으로.

DigitalOcean으로 돌아간 뒤 DNS 메뉴에서 구입한 도메인을 추가한다.
DNS Record 어쩌구가 보이는데, 사실 나도 저게 무엇인지는 잘 모른다. 구글의 힘을 빌려 공부해야겠다.

aseom.net으로 접속하니 잘 들어가진다.
WordPress 관리에 들어가 설정 > 일반 메뉴에서 블로그 주소와 사이트 주소를 구입한 도메인으로 지정한다.

SFTP로 서버에 접속해 티스토리에서 백업한 .xml 파일을 올려두고, WordPress의 TTXML 플러그인으로 임포트한다.

이사 끝!

DigitalOcean

싱가포르에 서버를 열었기에 속도는 나쁘지 않게 뽑아준다. 당연히 티스토리 만큼은 아니지만.
나만의 개인서버가 하나 생긴 셈이니, 블로그 뿐 아니라 다른 용도로도 활용할 생각이다.
내가 이 일을 해냈다는 생각에 괜한 뿌듯함도 들고, 아무튼 좋다.
뼛속까지 공돌이인 나에게 좋은 장난감이 생겼으니, 이제 남은 것은, 개인 서버와 개인 WordPress 블로그를 마음껏 괴롭히는(?) 것.

DigitalOcean을 세 달 간 이용해 보고, 도쿄에 데이터센터가 위치한 Linode로 옮길지 말지를 결정해야겠다.
마음같아서는 AWS를 이용하고 싶지만, 아직 AWS 관련해서는 뭔 소리인지 하나도 알아들을 수가 없기에…
지금으로써는, ‘blog.aseom.net’, ‘file.aseom.net’ 등으로 서브도메인을 만들고, 각각 특정 폴더로 포워딩 시킨 뒤, 서버를 여러가지 용도로 활용하는 것이 소박한 목표이다.

WordPress

WordPress는 그야말로 신세계.
고퀄리티의 테마는 물론이고, 플러그인도 다양하다.
설치형이니 만큼 모든 것이 자유롭고, 마치 티스토리에서 ‘탈출’한 것 같은 느낌이다.
아직 WordPress의 수많은 기능들을 모두 경험해보지는 못했지만, 각종 기능을 체험하게 될 앞으로가 기대된다.
관리 페이지도 깔끔하게 구성되어 있고, 공식 앱을 iPhone에 설치해보니, 아예 HTML 작성이 기본으로 되어 있다.

내 블로그와 잘 맞아 떨어지는 테마를 선택하고, 블로그를 취향대로 뜯어고칠 일이 남았다. 이를테면, 소셜 댓글 시스템이라든지 나눔바른고딕 웹폰트 적용같은거.
아무쪼록, 나의 블로그를 만족스러운 마음으로 계속 이어갈 수 있으면 좋겠다.

그리고…

눈치 빠른 사람이라면 벌써 알아챘겠지만, ‘Design by ASeom’에서 ‘Blog by ASeom’으로 블로그 이름이 달라졌다.
그래픽 디자인을 위주로 블로그를 꾸려갈 생각이었던 초심과는 달리, 다른 내용, 특히 IT관련 내용이 아주 많아졌기 때문.
