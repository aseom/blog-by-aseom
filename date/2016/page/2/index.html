[WP] 어긋난 첨부 이미지 주소 모두 바로잡기

2016년 1월 2일 by ASeom Han

블로그 주소가 바뀌는 등의 경우에,
이전에 작성한 글의 첨부 이미지 링크들이 엉뚱한 곳을 가리키게 되는 경우가 있다.

‘http://aseom.net/wp-content/…’로 업로드된 이미지들이
‘http://blog.aseom.net/wp-content/…’로 옮겨졌다고 치자.

글 하나하나 열어서 이미지 태그를 수정하는 건… 말이 안되고…
MySQL 명령어 하나로 해결 가능. REPLACE 함수를 이용하면 된다.

phpMyAdmin에서 워드프레스 데이터베이스의 ‘wp_posts’ 테이블을 열고,
SQL 탭으로 이동해 다음 명령어를 실행한다.

UPDATE wp_posts SET post_content = REPLACE(post_content, '기존 문자열', '바꿀 문자열');

본인은 나중에도 블로그 주소가 바뀔 경우를 대비해,
절대 주소로 wp-content 폴더를 가리키는 것을 상대 주소로 바꿔줬다.

UPDATE wp_posts SET post_content = REPLACE(post_content, 'http://blog.aseom.net/', '/');
