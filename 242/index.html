취미로 코딩하는 컴덕의 블로그

블로그에 웹폰트 적용

2014년 10월 7일 by ASeom Han

블로그의 폰트를 ‘Apple SD 산돌고딕 Neo’로 지정해 두었지만, 웹폰트가 아니다 보니 애플 기기에서 볼때만 내가 원하던 블로그 디자인이 나온다.
국내에 Mac이나 iOS 기기 사용자가 그리 많은것도 아니고, 아쉬움이 많았다.

고심 끝에 웹폰트 사용을 결정하고, 산돌고딕 Neo 시리즈와 비교해도 충분히 멋지다는 느낌을 받을 수 있는 ‘나눔바른고딕’을 선택했다. 무료이고, 어떤 용도이든 사용이 자유롭다는게 장점.
하지만 얇은 굵기가 없어서 웹폰트 사용을 포기하게 될 ‘뻔’ 했다.

이를 지켜보기라도 한 듯, 네이버가 ‘얇은 나눔바른고딕’을 추가로 공개했고, 이걸 누군가가 웹폰트로 만들어 주기만을 마냥 기다릴 수는 없으니 직접 해보기로 결심했다.

슬슬 작업 시작

먼저 네이버의 ‘나눔글꼴‘ 소개 페이지에서 나눔바른고딕과 얇은 나눔바른고딕을 내려받는다.

‘Google Fonts‘에서 ‘Roboto‘를 내려받는다.

굳이 다운로드 받지 않고 구글 서버에 존재하는 웹폰트 파일의 링크만 블로그 스킨에 추가해주어도 되지만, 기왕이면 웹폰트 파일을 스킨에 업로드하는게 로딩 속도 면에서 이득이라 생각되었다.
국내 인터넷 환경에서, 웹폰트를 구글 서버에서 받아오는 것 보다 티스토리 서버에서 받아오는 것이 더 빠를테니까.

필요한 폰트 스타일 선택

나눔바른고딕의 폰트 스타일, 총 4개다.

Roboto의 폰트 스타일, 총 12개다.

나눔바른고딕은 굵기별로 4개의 폰트 스타일을, Roboto는 굵기별로 6개, 이탤릭까지 합하면 총 12개의 폰트 스타일을 가지고 있다.
이 중 내 블로그에 필요한 굵기는 Light, Regular, Bold. 이 셋을 제외한 모든 파일을 삭제.

나눔바른고딕과 Roboto는 같은 폰트 스타일에서도 굵기에 약간 차이가 있었고, 육안으로 봤을때 비슷한 것으로 선택했다.
나눔바른고딕은 Light, Regular, Bold. Roboto는 Light, Regular, Medium.

폰트 용량 줄이기

나눔바른고딕은 유니코드 기반 한글 11,172자를 모두 지원하는 글꼴인 만큼, 용량이 상당하다.
용량을 줄이는 작업 없이 그냥 웹폰트로 사용한다면 로딩 속도도 느리고, 모바일에서 접속시 피같은 데이터를 낭비하게 될 것이 불보듯 뻔하다.
다소 번거롭더라도 용량을 줄일 방법을 찾아야 한다.

무료로 사용할 수 있는 오픈 소스 폰트 편집 프로그램인 ‘FontForge‘를 이용해 불필요한 문자를 삭제하는 것이 가장 바람직한(?) 방법.
영문 폰트는 Roboto를 사용할 것이므로 한글을 제외한 모든 문자 – 영문 알파벳, 한자, 기호, 특수문자 등 – 를 과감히 삭제했다.


불필요한 문자를 삭제한 후 대폭 줄어든 용량

Roboto는 특별히 수정을 가하지 않음.

변환하기

이제 웹폰트로 변환하는 과정만이 남았다.

‘font2web‘이라는 사이트에서 손쉽게 변환 가능.
OTF 또는 TTF 파일을 업로드하면 TTF, OTF, EOT, WOFF, SVG, CSS와 데모용 HTML 파일까지 한번에 생성해준다.

변환되어진 파일 중, SVG는 딱히 불필요할 것이라 여겨지고, CSS는 나중에 직접 작성할 것이므로 삭제했다.

CSS 작성

웹폰트를 불러오도록 하는 CSS 파일을 작성해야 한다.

‘webfont.css’

font-weight를 따로 지정해 주었다. 폰트 자체의 굵기와 관계없이 여기서 지정한 값으로 font-weight가 정해진다.
웹폰트를 블로그에 적용한 후 티스토리 스킨의 ‘style.css’에서


font-face: Roboto, font-weight: 300;

을 지정한다면 Roboto Light를 불러오게 되는 것.

웹폰트 CSS 작성과 관련된 자세한 정보는 여기를 참고.

끝

이제 결과물을 감상한다.

나눔바른고딕의 라이선스는 ‘SIL Open Font License 1.1‘, Roboto는 ‘Apache License 2.0‘이다.
블로그 어딘가에 폰트 라이선스를 따로 고지해 두어야 할 듯.
